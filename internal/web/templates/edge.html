Edge routes

Helmsman owns the public edge (HTTPS + automatic certificates). You add a route — a hostname → an app's internal endpoint — and Helmsman renders and applies the whole edge config for you. The edge can never reach the control plane (control-plane ports are refused).

Advanced — extra config (optional)

For the rare case the per-app routes above aren't enough, you can add extra vhosts, headers, or matchers as a JSON array of routes. This is an additive overlay: it can only add on hostnames you own — it can never redefine or shadow a route Helmsman manages, reach the control plane, or set forwarding headers. Anything outside the safe, typed subset (e.g. running a command, serving files, custom TLS) is rejected, not merged. Most operators never need this.
