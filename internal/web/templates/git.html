Connect with GitHub (one-click repo picker — not set up yet)

Want to authorize once and pick a repo from a list (like Netlify), with the deploy key installed for you? Set up a GitHub OAuth app once (Settings → Developer settings → OAuth Apps). The callback URL must match how you reach this dashboard: http://localhost:9000/github/callback over an SSH tunnel (no public domain needed), or https://<admin.hostname>/github/callback once you've set an admin domain. Then add github.client_id + github.client_secret to /etc/mooring/config.yaml and systemctl restart mooring (GitHub credentials are read only at startup, so a reload won't pick them up) — the Connect with GitHub button appears here. See the GitOps guide.

Mooring reads a mooring.yaml from your repo and generates the
        compose (and the Dockerfile, from each service's build:) — you never commit a
        docker-compose.yml. The app's name (slug) comes from the file's
        metadata.slug, so there's nothing to type here. If your repo holds several
        mooring files — mooring.yaml plus variants like mooring.staging.yaml or
        mooring.prod.yaml — you'll pick which one to deploy next (each is its own app).

      
      {{else}}
      
Mooring reads this app's mooring file at the deployed commit and generates the
        compose. The file in your repo is the single source of truth for the app's shape —
        edit it there and deploy; the dashboard shows it (and lets you set secret values, config files, and
        run/restart/scale).

      {{end}}
      

        Starter mooring.yaml — copy into your repo root
        
apiVersion: mooring/v1
kind: App
metadata:
  slug: myapp                # lowercase; the app's name
spec:
  compose:
    services:
      api:
        build:               # build from THIS repo (omit + use `image:` to pull instead)
          language: auto     # auto | node | python | go | ruby | php | static
        ports:
          - internal: 3000
        env:
          LOG_LEVEL: info
          JWT_SECRET: { secret: JWT_SECRET }   # value set in the dashboard, never in YAML
      cache:
        image: redis:7       # an off-the-shelf image service
  secrets:
    - name: JWT_SECRET       # declare the NAME here; set its VALUE in the dashboard
  edge:
    routes:
      - hostname: myapp.example.com
        service: api
        port: 3000           # Mooring terminates TLS and proxies to api:3000
